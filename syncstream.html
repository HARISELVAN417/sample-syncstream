🔧 Technical Architecture

SyncStream uses a combination of modern web technologies to deliver synchronized audio streaming:

Core Technologies

WebRTC WebSocket Web Audio API MediaStream API Service Workers

How It Works

1. Host Setup

The host device captures audio from the current video/movie playing and encodes it into chunks using the Web Audio API.

2. Room Broadcasting

A WebSocket server maintains real-time communication with all connected clients. Audio chunks are broadcast with precise timestamps.

3. Client Synchronization

Each client receives the stream, buffers chunks, and uses adaptive timing to sync playback within <1ms. Network jitter is compensated automatically.

4. Feedback Loop

Clients send heartbeat packets with latency & buffer metrics. The host monitors sync health and adjusts bitrate if needed.

Audio Pipeline

HOST → Capture (AudioContext) → Encode (PCM/Opus) → Timestamp → WebSocket Broadcast CLIENT → WebSocket Receive → Buffer (adaptive) → Decode → Web Audio API → Speaker Output
